🔷 2️⃣ OS 5 PADRÕES OBRIGATÓRIOS DE COMPOSIÇÃO (VINCULANTE)
Todo componente da Fase 3 DEVE implementar:
2.1 `inheritAttrs: false` e Repasse Explícito
- Declarar `inheritAttrs: false` no `defineOptions`.
- Repassar `$attrs` explicitamente para o nó raiz correto via `v-bind="$attrs"`.
2.2 Propagação de Estado via `provide/inject` Tipado
- Usar `provide` no componente raiz para expor estado global (ex: `disabled`, `readonly`).
- Usar `inject` nos componentes filhos para consumir esse estado.
- NUNCA usar prop drilling excessivo.
2.3 CSS Variables como Canal de Comunicação
- Propagar propriedades visuais (ex: `brand`) via atributos de dados (`data-*`) no elemento raiz.
2.4 Proibição Absoluta de `:deep()` para Layout
- O layout interno deve ser controlado por classes no wrapper do componente pai.
- NUNCA injetar CSS nos componentes filhos usando `:deep()`.
2.5 Slots Dinâmicos e Tipados
- Usar geração dinâmica de slots (`v-for` com `<slot :name="...">`) para repassar áreas de conteúdo.

🔷 3️⃣ PREVENÇÃO DE ERROS E DARK MODE
3.1 Nunca usar tokens inexistentes
- Validar contra DSS_TOKEN_REFERENCE.md.
3.2 Dark Mode e Contraste
- Garantir que todas as cores de fundo e texto tenham contraste adequado (WCAG AA) em ambos os temas.
- Evitar cores primárias fixas em fundos grandes; preferir `--dss-surface-muted` ou `--dss-surface-subtle`.
3.3 Acessibilidade (ARIA)
- O componente raiz deve gerenciar os atributos `aria-*` globais.
- Garantir que o foco flua logicamente e não fique preso em elementos desabilitados.
